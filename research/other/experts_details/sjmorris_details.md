🦖 We got the Chrome dino game running inside HubSpot.

Not in a new tab. Not in some external app. Inside an actual HubSpot card, on a live record.

A couple of weeks ago, Carter McKay shared a cool trick: HubSpot's <Image /> component will happily render an SVG... which means you can draw your own interface instead of being stuck with the default building blocks.

Now that Carter's shown us how to walk, it's time for the HubSpot community to run. So… we started with the Dino game.

How it actually works:

→ Every frame of the game is just an image — the dino, the cacti, the ground, drawn as an SVG.

→ We redraw that image ~20 times a second and swap it in. That rapid swap is the animation.

→ Controls are plain HubSpot buttons — plus a state-controlled input field, so you can use Space + W/S on your keyboard too.

No iframe. No plugins. No hacks. Just standard HubSpot components, used in a slightly unreasonable way.

But here's why this matters (and not just for time waste):

Most HubSpot portals look the same — because everyone's using the same handful of stock components. This breaks that ceiling. If we can run a playable game inside a card, we can build anything in there: custom dashboards, visual schedulers, branded tools, interactive reports. There is no more limits.

Huge thanks to Carter McKay for the spark. 🙏

👇 Use Space + W/S or the on-screen buttons and run it straight in your portal.
